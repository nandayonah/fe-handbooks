Git Workflow

  • Buat 2 branch utama yakni, develop dan master

  • Jadikan branch develop sebagai branch default

  • Di dalam tim terdapat 1 ketua (org yg dipercaya) yg mengelola branch master dan develop

  • Setiap task yang dikerjakan menggunakan branch khusus.

    • fitur: feature/register
    • bug: bugfix/fix-userinfo-undefined
    • hotfix: hotfix/improvement-styling-profile

  • Ketika akan commit sebaiknya menggunakan kalimat perintah apa yg dilakukan tidak terlalu general tidak terlalu spesifik:

    • Mengerjakan halaman register
    • Implementasi validasi dan api login

  • Alur Merge Request: branch-name --> Merge Request --> develop -> master (semua nya sudah jalan)

  • Diusahain code dari develop ke master itu kalo semuanya sudah jalan

  • Semua fitur yg dibuat dikumpulin di develop dulu

  • Ketua menjaga merge request, ingetin ke anggota jgn MR kalo fitur nya blum jalan atau kode nya masih belum bener

Clone

  • Open terminal git clone [URL]

Branching

  • After creating branch in GitLab, in your projects folder open in terminal and type following command: git fetch && git checkout [BRANCH_NAME]

Flow Creating Merge Request

  • Create a new branch for something that you work for.
  • Git Fetch and Git Checkout in your computer
  • Do some commits
  • Push git push origin feature/implement-login
  • Create Merge Request on GitLab

  • When the Merge Request is OK the leader could merge the request.

  • If Conflicts happens

    • git checkout develop then git pull origin develop to get latest code
    • back to your working branch, git checkout feature/implement-login
    • merge the develop branch git merge develop
    • solve the conflict
    • git add . then git commit then close the file to accept the changes
    • repush your work git push origin [BRANCH_NAME]
    • go to your Merge Request
    • merge
